##  你了解 Java 线程池的原理吗？

线程池是一种池化技术，用于预先创建并管理一组线程，避免频繁创建和销毁线程的开销，提高性能和响应速度。

关键配置：核心线程数 最大线程数 线程存活时长、线程存活时长单位、工作队列、拒绝策略。

public ThreadPoolExecutor()





具体工作原理：

1. 默认情况下线程不会预创建，只有当任务提交之后才会创建线程。
2. 当核心线程数满了之后，把任务加入到任务队列中。
3. 如果任务队列也满了的情况下，才会另外创建线程。
4. 如果另外创建的线程达到了最大线程数，就停止创建线程。
5. 如果还有任务不断加入，执行拒绝策略。默认的拒绝策略就是拒绝后续的任务。
6. 如果线程空闲的时长超过了设定的线程存活时长，并且当前的线程数量大于最大的核心线程数，就会销毁线程，直到线程数量等于核心线程数量。



- 可以通过prestartAllCoreThreads = true 预创建核心线程，默认为false。
- 销毁线程可以通过allowCoreThreadTimeOut = true 回收核心线程，默认为false。



工作队列类型

- SynchronousQueue 不存储任务，直接将任务提交给线程。
- LinkedBlockingQueue 链表结构的阻塞队列，大小无限。
- ArrayBlockingQueue 数组结构的有界阻塞队列。
- PriorityBlockingQueue 带优先级的无界阻塞队列。

   用于控制访问资源的线程数，可以用来实现限流或者访问控制。

   让一组线程到达一个共同的同步点，一起去执行。常用于分阶段任务执行。

   生产者线程将元素放入队列，消费者线程从队列中取元素，队列为空时消费者线程阻塞。





## 什么是 Java 的 CAS（Compare-And-Swap）操作？

CAS是一种硬件级别的原子操作，比较内存中的某个值是否为预期值，如果是，则更新为新值，否则不修改。



工作原理：

1. 比较：CAS检查内存中的某个值是否与预期值相等
2. 交换：如果相等，则将内存中的值更新为新值
3. 失败重试：如果不相等，说明有其他线程修改了该值，CAS操作失败，一般会利用重试，直到成功。





CAS的优缺点：

优点

- 无锁并发：CAS操作不使用锁，因此不会导致线程阻塞，提高了系统的并发性和性能。
- 原子性：CAS操作是原子的，保证了线程的安全。

缺点

- **ABA问题**：在CAS操作中，如果一个变量值从A到B，又变回A，CAS无法检测这种变化，可能导致错误。
- 自旋开销：CAS操作通常通过自旋实现，可能导致CPU资源浪费，尤其高并发下。
- 单变量限制：CAS操作仅适用于单个变量的更行，不适用于涉及多个变量的复杂操作。



## 说说 AQS 吧？

AQS   AQS就是起到了一个抽象、封装的作用，将一些排队、入队、加锁、中断等方法提供出来，便于其他相关JUC锁的使用，具体的加锁实际、入队实际等都需要实现类自己控制。



它主要通过维护一个共享状态（state）和一个先进先出的等待队列，来管理线程对共享资源的访问。

state用volatile修饰，表示当前资源的状态。比如在独占锁中，state为0表示未被占用，为1表示已被占用。



当线程尝试获取资源失败的时候，会被加入到AQS的等待队列中。这个队列是一个变体的CLH队列，采用双向链表结构，节点包含线程的引用、等待状态以及前驱和后继节点的指针。



AQS常见的实现类有ReentrantLock/CountDownLatch/Semaphoreb

## Synchronized 和 ReentrantLock 有什么区别？

1. Synchronize  Java关键字，可以用于变量、方法、类上，对当前的目前进行一个临时的加锁。实现基本的同步机制，不支持超时，非公平，不可中断，不支持多条件。
2. ReentrantLock 是JUC类库提供的，在JDK1.5中引入，支持设置超时时间，可以避免死锁，比较灵活，并且支持公平锁，可中断、支持多条件判断。



ReentrantLock需要手动解锁，而Synchronize不需要，它们都是可重入锁。

一般情况下Synchronize就足够了，而ReentrantLock比较灵活，支持的功能也比较多，所以复杂的情况用ReentrantLock。



### 可重入锁

指的是同一个线程在持有某个锁的时候，可以再次获取该锁而不会发生死锁。





## Java 中 volatile 关键字的作用是什么？



1. 变量全局可见性

    在并发中，某个线程对带了volatile关键字的变量进行了修改，对其他的线程来说是可见的，数据更新是实时的。

2. 防止指令重排序

   （1. 先声明 2. 实例化对象 3. 关联对象与声明的变量）

​	防止类实例化顺序改变。避免未完全初始化的对象被使用。经典之单例双重检锁模式。





newScheduledThreadPool 是 java.util.concurrent.Executors 类提供的工厂方法，用于创建支持定时或周期性任务调度的线程池。其底层实现基于 ScheduledThreadPoolExecutor，适合需要延迟执行或重复执行任务的场景。



newWorkStealingPool 是Java 8中引入的一个新型线程池，它基于ForkJoinPool实现，并采用了“工作窃取”（Work-Stealing）算法。这种线程池特别适用于可并行化且计算密集型的任务，能够充分利用多核CPU资源，提高任务执行效率。

在newWorkStealingPool中，每个线程都维护一个自己的任务队列（双端队列Deque）。当线程执行完自己队列中的任务后，它会尝试从其他线程的队列中“窃取”任务来执行，从而实现负载均衡。这种算法能够减少线程间的竞争，提高系统的整体性能。
